📖 What is CNV HealthCrew AI?

CNV HealthCrew AI is an internal tool built by the Performance Engineering team to automate cluster health monitoring and root cause analysis for our OpenShift/KubeVirt environments. It saves us hours of manual investigation by automatically detecting issues, matching them to known Jira bugs, and providing actionable recommendations.

⏱️

Save Time

Get instant root cause analysis instead of spending hours digging through logs and pods

🔍

Find Issues Fast

🧠

Learn from Jira

AI matches failures to known bugs and suggests new health checks from recent Jira issues

🚀 Key Innovations

🧠 Pattern-Based RCA

The system learns from multiple sources:

  • 🎫 Jira Bugs - Discovers new tests from bug reports
  • 📧 Email - Learns from team discussions & alerts
  • 🌐 Web - Searches docs, forums & knowledge bases
  • 🎯 Patterns - Recognizes recurring issues
  • ♾️ Never stops - Gets smarter with every run

⚡ Performance Engineering

Built for Performance Engineers:

  • 📊 Resource profiling - CPU, Memory, I/O per node
  • 🔥 Bottleneck detection - Find hotspots instantly
  • 📈 Trend analysis - Track performance over time
  • ⚠️ Threshold alerts - Proactive warnings at 85%+

▶️ Running Builds

A "build" is a single execution of health checks against your cluster. Here's how to run one:

🚀 Quick Build

1 Click "New Build" in the navigation bar, or use the "Quick Build" sidebar link
2 Quick Build runs with all checks pre-selected - just click "Run Now"

🔧 Configured Build

1 Go to "New Build" → Fill in the configuration form
2 Run Name (optional) - Give your run a descriptive name like "Pre-upgrade check" or "Scale test validation"
3 Target Server - Select the target host or leave empty for local kubeconfig
4 Select Options - Enable Jira Integration, Email Report, etc.
5 Choose RCA Level - Checks Only, Bug Digging, or Full RCA
6 Select Health Checks - Use presets or pick individual checks
7 Click "Run Now" - Build starts immediately and redirects to dashboard

📊 Build Results

Status Meaning
✅ Healthy All checks passed, no issues found
⚠️ Issues Found Some warnings detected, review recommended
❌ Failed Critical issues or errors encountered
⏹️ Stopped Build was manually stopped

⚙️ Build Options

🔬 Root Cause Analysis Levels

Level Description Speed
📋 Checks Only Run health checks without any analysis. Just collects data and reports findings. ⚡ Fastest
🐛 Bug Digging Match failures to known Jira bugs using the built-in KNOWN_ISSUES database. No external API calls. ⚡ Fast
🔍 Full RCA Runs targeted investigation commands per issue type, matches against known Jira bugs, analyzes logs, suggests specific fixes. 🐢 Slower (10-30s/issue)

📚 Data Sources for Bug Digging

Source Description Flag
🎫 Jira Bugs Search Jira for matching bugs in CNV, OCPBUGS, ODF projects --rca-jira
📧 Email Search Search Gmail for related discussions, alerts, and threads --rca-email
🌐 Web Search Search documentation, forums, and knowledge bases --rca-web

📤 Additional Options

📤 Email Report Send the HTML report via email after completion

📊 Alert Thresholds

Thresholds determine when resources are flagged as unhealthy. You can customize them per-run or change the defaults in Settings.

🎚️ Available Thresholds

Threshold Default Description
🔥 CPU Warning 85% Nodes with CPU usage above this are flagged as high CPU
💾 Memory Warning 80% Nodes with memory usage above this are flagged
💿 Disk I/O Latency 100ms Disk operations taking longer trigger a warning
🗄️ etcd Latency 100ms etcd latency above this is CRITICAL
📦 Pod Density 50 pods/node Warns when nodes have too many pods scheduled
🔁 Pod Restarts 5 Pods with more restarts than this are flagged
🖥️ virt-handler Memory 500Mi virt-handler pods using more memory are flagged (CNV-66551)

⚙️ Customizing Thresholds

  • Per-run override: On the New Build page, check "Use Custom Thresholds" and adjust values
  • Change defaults: Go to Settings → Alert Thresholds

🎫 Jira Integration

The tool can scan Jira for recent bugs and suggest new health checks based on patterns found.

🔍 How It Works

1 Enable Jira Integration - Check "Jira Integration" on the New Build page
2 Scan for Suggestions - Click "Scan Jira for New Test Suggestions"
3 Review Modal - A modal appears with suggested health checks from recent bugs
4 Accept or Reject - Review each suggestion and click Accept ✅ or Reject ❌
5 Accepted checks are added to future health check runs

🎯 Suggestion Modal Fields

Field Description
Check Name Suggested name for the new health check
Jira Key Link to the source Jira bug (clickable)
Priority Bug priority (Critical, Major, Minor, etc.)
Description What the check will monitor
Matched Keywords Keywords that triggered the suggestion
