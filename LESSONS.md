# Lessons Learned

Maintained at the end of sessions based on human feedback. All agents read this file before producing artifacts or committing code.

## How This File Works

- Lessons are generic, transferable patterns - not code-specific details.
- New lessons are proposed by the agent; the user accepts or rejects them.
- If a new lesson contradicts an existing one, the newer lesson supersedes it - mark the old entry with `[SUPERSEDED by <date>]`.
- Each lesson follows the format: `[Category] Pattern observed -> What to do differently -> Why it matters`
